| Directive Category | Configuration Parameter | Engineering Rationale and Token Implication |
| --- | --- | --- |
| **System Persona** | Role: Chief Surgical Architect | Forces the model to prioritize precise, highly targeted modifications over destructive, file-wide rewrites. This role mandates an artifact-driven approach to complex interventions, treating the codebase as a living system requiring surgical precision, thereby reducing wasted exploratory tokens. |
| **Operational Protocol** | Mode: Think-Act-Reflect | Requires the agent to generate a chain-of-thought sequence enclosed in `<thought>` tags prior to executing any destructive commands or file modifications. This mimics the cognitive architecture of advanced models, ensuring logical consistency before compute is expended on code generation. |
| **Security Invariants** | Constraint: Terminal Auto-Execution Limits | Mandates that the terminal execution policy remains in Auto or Off mode for destructive actions. It explicitly prevents the autonomous execution of DELETE, DROP, or recursive directory removal commands without explicit human verification, serving as a critical safeguard against hallucinated operations. |
| **Context Management** | Protocol: Progressive Disclosure | Forbids the agent from attempting to ingest massive directories simultaneously. Enforces the use of localized search tools and recursive summarization to maintain a sustainable token footprint, fundamentally preventing context saturation. |
| **Artifact Generation** | Requirement: Artifact-First Execution | Dictates that all multi-file refactoring or new feature generation must be preceded by an implementation plan written to `artifacts/plan_[task_id].md`. This establishes a trail of evidence for auditing and allows human intervention before costly implementation begins. |
| **Stylistic Baselines** | Constraint: Code Generation Standards | Prohibits the use of inline styling (except for dynamic computational values) and enforces strict semantic markup and established naming conventions (e.g., BEM methodology). This ensures that generated code meets senior-level production standards without requiring constant human refactoring. |
| **Agent Routing** | Constraint: Mandatory Skill Invocation | Instructs the global agent to defer to specific modular skills for highly sensitive operations. For example, dictating that any database modification must route through a "Safe-Migration" skill, bypassing raw SQL terminal inputs entirely. |
